# Phase 2.5 Documents

Phase 2.5 is the Toronto static Green P / public parking discovery bridge release.

This phase starts only after Phase 2 is accepted or frozen. It keeps the app local-first and adds only verified static public parking discovery. It must not introduce backend, login, cloud sync, analytics, ML, payment, real-time availability, community features, continuous background location, or the old History list.

## Current Authoritative Files

- `PHASE2_5_ROADMAP.md` - product scope, feature order, non-goals, and exit criteria.
- `PHASE2_5_ARCHITECTURE_REVIEW.md` - static catalog architecture, source boundaries, and risks.
- `PHASE2_5_GREENP_SOURCE_REVIEW.md` - source/licence/data validation checklist that must pass before visible Green P UI.
- `PHASE2_5_SELF_TEST.md` - Clawdbot Phase 2.5 self-test runbook and reporting instructions.
- `PHASE2_5_RELEASE_CANDIDATE_CHECKLIST.md` - final feature-freeze and release readiness checklist.
- `handoffs/` - development-to-test handoff notes for Phase 2.5 feature slices.

## Phase Rule

If no official or clearly reliable static Green P/public parking source is approved, Phase 2.5 stops at documentation, source review, import tooling design, and hidden prototype preparation. Do not ship visible Green P markers without source approval.
