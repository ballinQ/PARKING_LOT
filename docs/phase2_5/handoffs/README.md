# Phase 2.5 Handoffs

Development thread should add one handoff document here for each completed Phase 2.5 feature slice.

Use this naming pattern:

`YYYYMMDD_HHMM_feature_name_TEST_HANDOFF.md`

Each handoff should describe:

- feature goal
- files changed
- new behavior
- behavior intentionally not changed
- source/data assumptions
- known risks
- notes for Test/QA
